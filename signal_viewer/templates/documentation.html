Signal Viewer Documentation

Getting Started

Installation

Clone the repository and install dependencies:

git clone <repo-url>
cd signal_viewer
pip install -r requirements.txt

Setup

Set the HDF5 data root directory:

export DATA_ROOT=/path/to/hdf5/files

Start the development server:

python -m signal_viewer.server.app

Open your browser and navigate to http://localhost:8000

Data Format

HDF5 Structure

Signal Viewer expects HDF5 files organized as follows:

DATA_ROOT/
├── serial_name/
│   ├── step_001/
│   │   └── batch_file.h5
│   │       ├── batch_001/
│   │       │   ├── signal_000
│   │       │   ├── signal_001
│   │       │   └── ...
│   │       └── batch_002/
│   └── step_002/
└── other_serial/

Signal Requirements

  • Signals stored as HDF5 datasets within batch groups
  • Each signal is a 1D float array (time series data)
  • Metadata attributes optional: units, sample_rate, description
  • Supported file extensions: .h5, .hdf5

Navigation

File Selection

The navigation bar at the top provides three cascading dropdowns:

  • Serial: Select a test serial or unit identifier
  • Step: Choose a test step (populated after serial selection)
  • File: Select a specific HDF5 file (populated after step selection)

Sidebar Navigation

The left sidebar displays available signals from the selected file:

  • Click a signal to view it
  • Signals are numbered (signal_000, signal_001, etc.)
  • Batch information displayed for context

Main Navigation

Top navigation menu links to main sections:

  • Viewer: View and analyze individual signals
  • Analysis: Statistical analysis and processing
  • Comparison: Compare multiple signals
  • Docs: This documentation

Viewer

Display Modes

The viewer supports multiple visualization modes:

  • Overlay Mode: Stack up to 5 signals on the same plot
  • Subplot Mode: Display each signal in a separate subplot

Signal Cards

Each loaded signal displays a card with:

  • Signal name and metadata
  • Basic statistics (min, max, mean, std)
  • Remove button to unload the signal
  • Visual indicator showing signal color in plot

Interactive Features

  • Hover Information: Display data values and coordinates on hover
  • Zoom: Click and drag to zoom into regions of interest
  • Pan: Hold shift and drag to pan across the signal
  • Reset View: Double-click to reset zoom/pan

Signal Limit

Up to 5 signals can be loaded simultaneously to maintain performance. Remove signals as needed to load new ones.

Analysis

Statistics

Detailed statistical analysis of loaded signals:

  • Descriptive Stats: Min, max, mean, standard deviation, median
  • Distribution: Histogram showing value distribution
  • Rainflow Cycle Counting: ASTM E1049 compliant cycle detection

Rainflow Cycle Counting

Identifies stress cycles in the signal data using the standard rainflow algorithm:

  • Detects local minima and maxima
  • Counts cycles and calculates range for each cycle
  • Useful for fatigue analysis and life prediction
  • Results displayed in summary table and histogram

Histogram

Visualizes the distribution of signal values:

  • Automatically determines appropriate bin sizing
  • Shows frequency of values in each bin
  • Supports overlay of multiple signal histograms

Comparison

Overlay Comparison

Display two signals on the same plot to compare waveforms directly. Useful for:

  • Comparing signals from different test conditions
  • Identifying differences in amplitude and phase
  • Visual alignment of features

X vs Y Correlation

Scatter plot showing the relationship between two signals:

  • X-axis: First signal values
  • Y-axis: Second signal values
  • Reveals nonlinear relationships and hysteresis

Combined Histogram

Overlay histograms of two signals for distribution comparison:

  • Side-by-side visualization of value ranges
  • Identify differences in statistical properties

Cross-Correlation

Measures similarity and time alignment between signals:

  • Detects time lags between signals
  • Quantifies correlation strength (-1 to +1)
  • Useful for signal synchronization analysis

Signal Chips

Extract and compare corresponding segments from two signals:

  • Select time range of interest
  • View zoomed-in details of matched segments

API Reference

Navigation Endpoints

  • GET /api/serials - List all available serials
  • GET /api/serials/{serial}/steps - List steps for a serial
  • GET /api/serials/{serial}/steps/{step}/files - List files in a step

Data Endpoints

  • GET /api/files/{file}/batches - List batches in a file
  • GET /api/files/{file}/batches/{batch}/meta - Get batch metadata
  • GET /api/files/{file}/batches/{batch}/signals/{index} - Get signal data

Analysis Endpoints

  • POST /api/analysis/stats - Calculate statistics
  • POST /api/analysis/fft - Compute FFT
  • POST /api/analysis/psd - Compute power spectral density
  • POST /api/analysis/filter - Apply signal filters
  • POST /api/analysis/correlation - Calculate cross-correlation
  • POST /api/analysis/trend - Analyze signal trends
  • POST /api/analysis/anomaly - Detect anomalies

Cache Endpoints

  • GET /api/cache/stats - Get cache statistics
  • POST /api/rescan - Rescan data directory

Keyboard Shortcuts

Plotting Controls

  • Click and drag - Zoom into region
  • Shift + Click and drag - Pan across plot
  • Double-click - Reset zoom/pan
  • Scroll wheel - Zoom in/out
  • Right-click - Context menu (save image, etc.)

Navigation

Configuration

Environment Variables

Configure the application using environment variables:

  • DATA_ROOT - Root directory containing HDF5 files (required)
    Default: ./data
  • DEBUG - Enable debug mode for development
    Default: False
    Usage: export DEBUG=True
  • HOST - Server host address
    Default: localhost
    Usage: export HOST=0.0.0.0
  • PORT - Server port number
    Default: 8000
    Usage: export PORT=8080
  • CACHE_MAX_MEMORY_MB - Signal cache size in MB
    Default: 512
    Usage: export CACHE_MAX_MEMORY_MB=1024
  • CACHE_CLEANUP_INTERVAL - Cache cleanup interval in seconds
    Default: 300

Example Configuration

export DATA_ROOT=/data/hdf5_signals
export DEBUG=True
export HOST=0.0.0.0
export PORT=8080
export CACHE_MAX_MEMORY_MB=1024

Troubleshooting

No Serials Appearing

Problem: Serial dropdown is empty after server starts.

  • Verify DATA_ROOT environment variable is set correctly
  • Check that HDF5 files exist in DATA_ROOT
  • Ensure directory permissions allow reading
  • Check server logs for errors: python -m signal_viewer.server.app

Signals Not Loading

Problem: Selected file shows no signals in sidebar.

  • Verify HDF5 file structure matches expected format
  • Check that signals are stored as datasets in batch groups
  • Use HDF5 viewer tool (e.g., HDFView) to inspect file structure
  • Check browser console for JavaScript errors (F12)

Out of Memory

Problem: "Cache limit exceeded" error when loading signals.

  • Increase CACHE_MAX_MEMORY_MB environment variable
  • Reduce number of loaded signals (max 5)
  • Close browser tabs and restart server
  • Check system memory availability

Slow Performance

Problem: Application is sluggish or unresponsive.

  • Reduce number of signals loaded simultaneously
  • Close other applications to free system resources
  • Check CPU usage in system monitor
  • Consider reducing plot update frequency in browser developer tools

Plot Not Updating

Problem: Plot remains blank or shows old data.

  • Refresh browser page (Ctrl+R or Cmd+R)
  • Clear browser cache and reload
  • Check browser console for JavaScript errors
  • Verify Plotly library loaded: check network requests in DevTools

CORS Errors

Problem: Browser console shows CORS errors.

  • Ensure server is running and accessible
  • Check that HOST and PORT match browser location
  • If using reverse proxy, verify CORS headers are set correctly
