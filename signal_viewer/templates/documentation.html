Signal Viewer Documentation

Getting Started

Installation

Clone the repository and install dependencies:

git clone <repo-url>
cd signal_viewer
pip install -r requirements.txt

Setup

Set the HDF5 data root directory:

export SIGNAL_VIEWER_DATA_ROOT=/path/to/hdf5/files

Start the development server:

python -m signal_viewer.server.app

Open your browser and navigate to http://127.0.0.1:8050

Data Format

Directory Structure

Signal Viewer expects HDF5 files in a three-level directory hierarchy:

SIGNAL_VIEWER_DATA_ROOT/
├── SN001/                      # serial (prefix SN*)
│   ├── p001_motor_test/        # folder_1 (pattern pXXX_*)
│   │   ├── run_nominal/        # folder_2 (any name)
│   │   │   └── data.h5
│   │   └── run_overload/
│   │       └── data.h5
│   └── p002_vibration/
└── SN002/

HDF5 Internal Structure

Each .h5 file contains one or more groups (batches) with dataset suffixes:

data.h5
├── GROUP_T0/
│   ├── GROUP_T0_V    (2D float64) signal values [M signals × N samples]
│   ├── GROUP_T0_TIM  (1D float64) epoch ms of first sample per signal [M]
│   ├── GROUP_T0_FRE  (1D float64) sampling frequency in Hz [M]
│   ├── GROUP_T0_SAM  (1D int64)   valid sample count per signal [M]
│   ├── GROUP_T0_N    (1D string)  signal names [M]
│   ├── GROUP_T0_UNI  (1D string)  signal units [M]
│   ├── GROUP_T0_ERR  (2D, optional — Type B only)
│   ├── GROUP_T0_SQI  (1D, optional — Type B only)
│   └── GROUP_T0_TLS  (1D, optional — Type B only)
└── GROUP_T1/
    └── ...

Signal Types

The reader supports two signal types, detected automatically per batch:

  • Type A (standard): signal datasets only. Truncated by the _SAM (n_samples) attribute. Time is constructed as t0 + i × (1000 / fs) ms.
  • Type B (extended): signal datasets alongside _ERR, _SQI, _TLS companion datasets. Also truncated by _SAM. Time constructed identically.

Each signal is a row in the _V matrix, truncated to _SAM samples. Signal names and units are read from the _N and _UNI datasets. Supported file extensions: .h5, .hdf5.

Navigation

File Selection

The navigation bar at the top provides three cascading dropdowns:

  • Serial: Select a test serial or unit identifier
  • Step: Choose a test step (populated after serial selection)
  • File: Select a specific HDF5 file (populated after step selection)

Sidebar Navigation

The left sidebar displays available signals from the selected file:

  • Click a signal name to load it into the viewer
  • Signals are grouped by batch (e.g. GROUP_T0, GROUP_T1)
  • Signal names and units come from the HDF5 metadata

Main Navigation

Top navigation menu links to main sections:

  • Viewer: View and analyze individual signals
  • Analysis: Statistical analysis and processing
  • Comparison: Compare multiple signals
  • Docs: This documentation

Viewer

Display Modes

The viewer supports two layout modes, selectable from the Layout dropdown:

  • Same Axis (Overlay): All signals share one set of axes. When a single signal is loaded the y-axis shows its units; with multiple signals it shows "Value".
  • Subplots: Each signal gets its own subplot with an independent y-axis labelled with the signal's units. All subplots share the same time range.

Switching layout mode while zoomed automatically reloads the full overview data so the signal is always displayed correctly.

Signal Cards

Each loaded signal displays a card in the header bar with:

  • Batch name and signal name (colour-coded to match the plot trace)
  • Live hover readout: value, timestamp, and estimated full-signal sample index
  • Point count ratio (displayed points / total samples)
  • Remove button (×) to unload the signal

Adaptive Zoom

The viewer uses an adaptive zoom system that re-fetches data from the server at higher resolution whenever you zoom in:

  • Zoom: Click and drag to select a region. After a short debounce, the server returns up to 5,000 points for just the visible window, giving pixel-level detail at any zoom depth.
  • Pan: Hold shift and drag to pan across the signal.
  • Reset zoom: Three ways to return to the full overview — the house icon (⌂) in the toolbar, the "Reset Zoom" button in the header, or double-clicking the plot.
  • Zoom indicator: While zoomed, a badge shows what percentage of the full signal is visible, and a "Refining…" label appears briefly while new data loads.

The server uses a two-pass MinMax-LTTB downsampling algorithm that preserves visual shape while keeping the point count low. A 5 % margin is fetched beyond the visible edges to avoid low-resolution strips when panning slightly.

Hover Crosshair

In subplot mode, hovering over any subplot draws a synchronised vertical crosshair across all subplots and updates every signal card with the interpolated value at that time.

Signal Limit

Up to 5 signals can be loaded simultaneously to maintain performance. Remove signals as needed to load new ones.

Analysis

Click a signal in the sidebar to run a full analysis. The page works with one signal at a time.

Statistics

Descriptive statistics for the selected signal:

  • Descriptive Stats: Min, max, mean, standard deviation, median
  • Distribution: Histogram showing value distribution
  • Rainflow Cycle Counting: ASTM E1049 compliant cycle detection

Rainflow Cycle Counting

Identifies stress cycles using the four-point rainflow algorithm:

  • Counts full and half cycles with range calculations
  • Percentile table showing range distribution
  • Range histogram with configurable bin count
  • Exceedance curves (range, cycle max, cycle min) for fatigue analysis

Value Histogram

Visualizes the distribution of the selected signal's values:

  • Automatically determines appropriate bin sizing
  • Shows frequency of values in each bin

Comparison

Click signals in the sidebar to assign them: first click = Signal A, second click = Signal B. Four comparison views are generated automatically:

Overlay Comparison

Both signals on the same time axis to compare waveforms directly. Useful for:

  • Comparing signals from different test conditions
  • Identifying differences in amplitude and phase
  • Visual alignment of features

X vs Y Correlation

Scatter plot showing the relationship between two signals:

  • X-axis: First signal values
  • Y-axis: Second signal values
  • Reveals nonlinear relationships and hysteresis

Combined Histogram

Overlay histograms of two signals for distribution comparison:

  • Side-by-side visualization of value ranges
  • Identify differences in statistical properties

Cross-Correlation

Measures similarity and time alignment between signals:

  • Detects time lags between signals
  • Quantifies correlation strength (-1 to +1)
  • Useful for signal synchronization analysis

API Reference

Navigation Endpoints

  • GET /api/serials - List all available serials
  • GET /api/serials/{serial}/steps - List steps for a serial
  • GET /api/serials/{serial}/steps/{step}/files - List files in a step

Data Endpoints

  • GET /api/files/{file}/batches - List batches in a file
  • GET /api/files/{file}/batches/{batch}/meta - Get batch metadata (signal names, units)
  • GET /api/files/{file}/batches/{batch}/signals/{index} - Get signal data. Supports optional query parameters:
    • downsample=N - Target point count (default: full signal, max 10,000). Uses MinMax-LTTB downsampling.
    • t_min=<epoch_ms> & t_max=<epoch_ms> - Time window slice. The server binary-searches the cached signal and returns only the matching segment. Combine with downsample for adaptive zoom.
    Response includes total_samples, t_start, t_end (full signal extent) and windowed (boolean) alongside the time and values arrays.

Analysis Endpoints

  • POST /api/analysis/stats - Descriptive statistics and rainflow cycle counting
  • POST /api/analysis/correlation - Cross-correlation between two signals
  • POST /api/analysis/trend - Polynomial trend fitting

Cache Endpoints

  • GET /api/cache/stats - Get cache statistics
  • POST /api/rescan - Rescan data directory

Configuration

Environment Variables

Configure the application using environment variables:

  • SIGNAL_VIEWER_DATA_ROOT - Root directory containing HDF5 files
    Default: <project>/data/
  • SIGNAL_VIEWER_HOST - Server bind address
    Default: 127.0.0.1
    Usage: export SIGNAL_VIEWER_HOST=0.0.0.0
  • SIGNAL_VIEWER_PORT - Server port number
    Default: 8050
    Usage: export SIGNAL_VIEWER_PORT=9000
  • SIGNAL_VIEWER_DEBUG - Enable debug mode (auto-reload)
    Default: true
    Usage: export SIGNAL_VIEWER_DEBUG=false
  • SIGNAL_VIEWER_VERBOSE - Log detailed tracebacks for every error
    Default: false
  • SIGNAL_VIEWER_CACHE_MB - Signal cache budget in MB
    Default: 500
    Usage: export SIGNAL_VIEWER_CACHE_MB=1024

Example Configuration

export SIGNAL_VIEWER_DATA_ROOT=/data/hdf5_signals
export SIGNAL_VIEWER_HOST=0.0.0.0
export SIGNAL_VIEWER_PORT=8050
export SIGNAL_VIEWER_CACHE_MB=1024

Troubleshooting

No Serials Appearing

Problem: Serial dropdown is empty after server starts.

  • Verify SIGNAL_VIEWER_DATA_ROOT environment variable is set correctly
  • Check that HDF5 files exist in the data root directory
  • Ensure directory permissions allow reading
  • Check server logs for errors: python -m signal_viewer.server.app

Signals Not Loading

Problem: Selected file shows no signals in sidebar.

  • Verify HDF5 file structure matches expected format
  • Check that signals are stored as datasets in batch groups
  • Use HDF5 viewer tool (e.g., HDFView) to inspect file structure
  • Check browser console for JavaScript errors (F12)

Out of Memory

Problem: "Cache limit exceeded" error when loading signals.

  • Increase SIGNAL_VIEWER_CACHE_MB environment variable
  • Reduce number of loaded signals (max 5)
  • Close browser tabs and restart server
  • Check system memory availability

Slow Performance

Problem: Application is sluggish or unresponsive.

  • Reduce number of signals loaded simultaneously
  • Close other applications to free system resources
  • Check CPU usage in system monitor
  • Consider reducing plot update frequency in browser developer tools

Plot Not Updating

Problem: Plot remains blank or shows old data.

  • Refresh browser page (Ctrl+R or Cmd+R)
  • Clear browser cache and reload
  • Check browser console for JavaScript errors
  • Verify Plotly library loaded: check network requests in DevTools

CORS Errors

Problem: Browser console shows CORS errors.

  • Ensure server is running and accessible
  • Check that SIGNAL_VIEWER_HOST and SIGNAL_VIEWER_PORT match browser location
  • If using reverse proxy, verify CORS headers are set correctly
